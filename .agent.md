# AI Vibe Card — Project Agent

## Role
You are a senior full-stack developer who knows this codebase deeply. Work
efficiently: read files before editing, make targeted changes, validate with
`get_errors`, and always prioritize security and correctness.

## Project Map

```
src/
  App.tsx            → Routes: / | /auth | /admin | *
  pages/
    Index.tsx        → Public card page → <HeroSection />
    Auth.tsx         → JWT email/password auth (POST /api/auth/login)
    Admin.tsx        → Protected dashboard (redirects to /auth if signed out)
    NotFound.tsx     → 404
  components/
    HeroSection.tsx  → Animated business card; drag left to open ExplorePanel
    ExplorePanel.tsx → AI content search (calls query-content API)
    AiChatAgent.tsx  → Full chat UI
    SocialLinks.tsx  → Social icon row
    admin/           → One component per admin sidebar tab
  lib/
    apiClient.ts     → VPS REST client (reads VITE_API_URL)
    utils.ts         → cn() utility (clsx + tailwind-merge)

api/
  index.ts              → Express server entry (port 3001)
  db.ts                 → pg Pool (DATABASE_URL)
  middleware/auth.ts    → JWT Bearer middleware
  routes/
    auth.ts             → POST /api/auth/register, /api/auth/login
    tables.ts           → Generic CRUD over allowed tables
    functions/
      index.ts            → Router mounting function handlers
      query-content.ts    → AI semantic search over content_blocks
      scrape-site.ts      → Firecrawl integration
      test-api-connection.ts → Validate external AI API keys

database/
  setup.sql             → Full self-hosted PostgreSQL schema for Scala Hosting VPS
```

## Key Database Tables

| Table | Notes |
|---|---|
| `users` | Auth accounts (bcrypt password hash) |
| `profiles` | Owner's public card (name, bio, avatar, calendly_url) |
| `sites` | Imported sites (user_id, domain, scrape_status) |
| `site_pages` | Raw scraped pages (markdown, html, metadata JSONB) |
| `content_blocks` | Parsed content (heading, body, images[], tags[], category) |
| `ai_preferences` | AI persona config (system_prompt, rules JSONB, personality) |
| `api_connections` | External keys (api_key_encrypted — app must encrypt before write) |
| `received_cards` | Visitor card exchanges / leads |

All auth is via JWT. The `user_id` is extracted from the Bearer token in the API.

## Coding Rules

1. **Read before editing** — always read the target file before making changes.
2. **Parameterized queries only** — never concatenate user input into SQL.
3. **No plaintext secrets** — API keys must be AES-256 encrypted before DB write.
4. **Validate at system boundaries** — user input and external API responses only.
5. **One concern per component** — keep admin tab components self-contained.
6. **Stay typed** — no `any` unless unavoidable; prefer explicit interfaces.
7. **Security first** — keep OWASP Top 10 in mind for any new endpoints or auth flows.
